### 1A. Subway Surfers — The Real Legal Picture

**Bottom line: Legally precarious. Widely tolerated. Not safe.**

Subway Surfers is owned by **Kiloo Games / SYBO Games** (Copenhagen, Denmark). Unlike Mojang/Microsoft, they have **no published creator policy, no Content ID whitelist program, and no monetization grant**. The reason millions of videos use it without being struck is a combination of:

1. **YouTube's "claimed, not struck" dynamic** — SYBO likely has Content ID set to *track* rather than *block*, meaning they harvest ad revenue from your video instead of striking it. You lose monetization silently.
2. **Volume immunity illusion** — "Everyone does it" doesn't mean it's legal. It means enforcement hasn't been priority yet.
3. **Fair use arguability** — The gameplay is transformative (background, not the focus), but this is an *affirmative defense* you have to litigate, not a pre-clearance.

**Has any channel been struck for Subway Surfers?**
- Confirmed pattern on Reddit (r/NewTubers, r/PartneredYouTube): Channels report **Content ID claims on ad revenue** rather than strikes. SYBO appears to use the revenue-share model of Content ID. The audio from the game (music/SFX) is the primary trigger.
- **Risk: Revenue claim = HIGH. Strike = MEDIUM-LOW. Termination = VERY LOW** (but non-zero for large channels).

```python
"""

OUTPUT_DIR = Path("backgrounds")
OUTPUT_DIR.mkdir(exist_ok=True)

NICHE_QUERIES = {

# ─── PEXELS ────────────────────────────────────────────────────────────────────
def search_pexels_videos(query: str, per_page: int = 5, min_duration: int = 5, 
                          orientation: str = "landscape") -> list[dict]:
    """Search Pexels video library."""
    headers = {"Authorization": PEXELS_API_KEY}
    params = {
        "query": query,
        "per_page": per_page,
        "orientation": orientation,
        "size": "medium",  # small | medium | large
    }
    resp = requests.get("https://api.pexels.com/videos/search", 
                        headers=headers, params=params, timeout=10)
    resp.raise_for_status()
    videos = resp.json().get("videos", [])
    # Filter by minimum duration
    return [v for v in videos if v.get("duration", 0) >= min_duration]

def download_pexels_video(video: dict, output_path: Path, 
                           preferred_quality: str = "hd") -> Optional[Path]:
    """Download best available quality from Pexels video files."""
    quality_order = ["uhd", "hd", "sd"]
    if preferred_quality in quality_order:
        quality_order.insert(0, quality_order.pop(quality_order.index(preferred_quality)))
    
    video_files = video.get("video_files", [])
    # Sort by quality preference then resolution
    for quality in quality_order:
        for vf in sorted(video_files, key=lambda x: x.get("width", 0), reverse=True):
            if quality in vf.get("quality", "").lower() or \
               quality in str(vf.get("link", "")).lower():
                url = vf["link"]
                ext = url.split("?")[0].split(".")[-1] or "mp4"
                filepath = output_path.with_suffix(f".{ext}")
                
                with requests.get(url, stream=True, timeout=30) as r:
                    r.raise_for_status()
                    with open(filepath, "wb") as f:
                        for chunk in r.iter_content(chunk_size=8192):
                            f.write(chunk)
                
                print(f"  ✅ Pexels [{video['id']}] → {filepath.name} "
                      f"({vf.get('width')}x{vf.get('height')})")
                return filepath
    return None

# ─── PIXABAY ───────────────────────────────────────────────────────────────────
def search_pixabay_videos(query: str, per_page: int = 5, 
                           min_duration: int = 5) -> list[dict]:
    """Search Pixabay video library."""
    params = {
        "key": PIXABAY_API_KEY,
        "q": query,
        "video_type": "film",  # film | animation | all
        "per_page": per_page,
        "safesearch": "true",
        "order": "popular",

                        params=params, timeout=10)
    resp.raise_for_status()
    hits = resp.json().get("hits", [])
    return [v for v in hits if v.get("duration", 0) >= min_duration]

    """Download best available quality from Pixabay."""
    videos_dict = video.get("videos", {})
    for quality in ["large", "medium", "small", "tiny"]:
        vf = videos_dict.get(quality, {})
        if vf.get("url"):
            url = vf["url"]
            filepath = output_path.with_suffix(".mp4")
            
            with requests.get(url, stream=True, timeout=30) as r:
                r.raise_for_status()
                with open(filepath, "wb") as f:
                    for chunk in r.iter_content(chunk_size=8192):
                        f.write(chunk)
            
            print(f"  ✅ Pixabay [{video['id']}] → {filepath.name} "
                  f"({vf.get('width')}x{vf.get('height')})")
            return filepath
    return None

# ─── MAIN BATCH DOWNLOAD ───────────────────────────────────────────────────────
```

```python
    cmd = base_cmd[:3] + effect_filters + base_cmd[3:] + [str(output_path)]
```

```bash
ffmpeg -loop 1 -i background.jpg -loop 1 -i foreground.png \
  -filter_complex \
    "[0:v]scale=4000:-1,crop=1920:1080:x='trunc(t*20)':y=0[bg]; \
     [1:v]scale=1920:1080,crop=1920:1080:x='trunc(t*40)':y=0[fg]; \
     [bg][fg]overlay=0:0[out]" \
  -map "[out]" -t 10 -r 30 -pix_fmt yuv420p parallax.mp4
```

### 7C. Particle Overlay System

```bash
# Rain overlay using noise + blur
ffmpeg -f lavfi -i nullsrc=s=1920x1080:r=30 \
  -vf "geq=lum='if(gt(random(1),0.998),255,0)':cb=128:cr=128,
       minterpolate=fps=30,
       motionblur=shutter_angle=180:samples=1,
       gblur=sigma=0.5" \

# Combine with your background:
ffmpeg -i background.mp4 -i rain_particles.mp4 \
  -filter_complex "[1:v]colorchannelmixer=aa=0.3[rain];[0:v][rain]overlay=0:0" \
  -t 10 rain_composite.mp4
```

### 7D. Film Grain + Blur + Vignette (Horror Combo)

```bash

  -vf "
    gblur=sigma=2,
    noise=alls=25:allf=t+u,

```

### 7E. Seamless Loop Creator

```bash
# Make any video loop seamlessly by crossfading end→start
DURATION=$(ffprobe -v quiet -show_entries format=duration -of csv=p=0 input.mp4)
FADE=1  # 1 second crossfade

ffmpeg -i input.mp4 \
  -filter_complex "
    [0:v]split[a][b];
    [a]trim=0:$(echo "$DURATION - $FADE" | bc),setpts=PTS-STARTPTS[main];
    [b]trim=$(echo "$DURATION - $FADE" | bc):$DURATION,setpts=PTS-STARTPTS[tail];
    [0:v]trim=$FADE:$(echo "$DURATION" | bc),setpts=PTS-STARTPTS[head];
    [tail][head]xfade=transition=fade:duration=$FADE:offset=0[out]
  " \
  -map "[out]" -c:v libx264 -crf 18 seamless_loop.mp4
```

### 7F. Slow-Mo Stretch (Turn 5s clip into 30s ambient)

```bash
# 6x slow-down using optical flow interpolation (minterpolate)

  -vf "minterpolate=fps=180:mi_mode=mci:mc_mode=aobmc:me_mode=bidir:vsbmc=1" \
  -r 30 -c:v libx264 -crf 18 slowmo_6x.mp4
```

## SECTION 8: BRAND CONSISTENCY — SAME BACKGROUND ACROSS PLAYLIST?

### The Decision Framework

**YES — Same visual style (not same clip):**
- Repeat the **color grade + filter recipe** across all videos
- Use **different clips** but from the same visual family
- Example: All horror videos use the desaturated/grain look, but different hallways each time

**NO — Don't reuse the same clip:**
- Viewer recognition kills the "immersive" quality fast (they notice the loop)
- YouTube's algorithm may flag repetitive content patterns
- Exception: Short ambient background cards (intros/outros) can be identical

**The playlist brand system:**

```
Playlist Visual Identity System:

├── Filter Recipe (fixed, apply to any background)

├── Background VARIETY (rotate from a library of 20-30 clips)
│   └── Download 10 clips/niche using the Python script above
│   └── Process all through the same filter recipe
│   └── Rotate randomly — viewers never see a repeat within 10 videos
│
└── UI Overlays (fixed: same text box style, same opacity gradient)
    └── Dark semi-transparent bottom bar: same every video
    └── Logo watermark: same corner, same opacity
```

**Answer: Same STYLE, same COLOR GRADE, different underlying footage.**
