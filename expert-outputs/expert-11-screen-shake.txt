SHAKE TYPE 1: LIGHT IMPACT (Minor Disturbance)
================================================================================

SCENARIO:
Small hit, minor collision, creature footstep, or environmental disturbance
that should be noticeable but not intrusive.

INTENSITY SPECIFICATIONS:
- Max displacement: 2-4 pixels
- Target typical: 3 pixels average
- Direction: Random XY offset per frame

DURATION:
- Total duration: 200-300 milliseconds
- Frame count: 6-9 frames @ 30fps
- Typical: 7 frames (233ms)

FREQUENCY:
- Vibration frequency: 15 Hz
- Frame-by-frame vibration: 2 cycles per 300ms
- Period: 67 milliseconds per cycle
- Characteristic: Rapid, zippy vibration

DISPLACEMENT PATTERN (Typical 7-frame example):
Frame 1 (0ms):   Offset +2px, +1px
Frame 2 (33ms):  Offset -3px, +2px
Frame 3 (67ms):  Offset +1px, -2px
Frame 4 (100ms): Offset -2px, -1px
Frame 5 (133ms): Offset +3px, 0px
Frame 6 (167ms): Offset -1px, +2px
Frame 7 (200ms): Offset +0px, -1px (damping)

DECAY FUNCTION:
- Decay type: Linear fade to zero
- Decay rate: 1/6 per frame (linear)
- Frame 1: 100% intensity (3 pixels)
- Frame 4: 50% intensity (1.5 pixels)
- Frame 7: 16% intensity (0.5 pixels)

RETURN SPEED:
- Type: Instant (no easing)
- Transition: Immediate calm return
- Perception: Noticeable snap back to baseline

BLEND MODE:
- Camera offset: Direct XY translation of camera matrix
- No rotation component
- Simple 2D displacement

PERFORMANCE PROFILE:
- CPU cost: 0.1ms per frame
- GPU cost: 0x (no GPU rendering cost)
- Memory: Negligible (only 2 float values)
- Frame rate impact: <1% of budget

USE CASES:
- Small punch/jab impact
- Creature footstep tremor
- Minor collision
- Door slam
- Small explosion nearby
- Arrow impact (non-critical)

EXAMPLE APPLICATION:
Scene: Rabbit jumping (Scene 2)
Trigger: Rabbit lands from jump
Intensity: 3 pixels
Duration: 200ms (6 frames)
Effect: Small tremor showing landing impact

SHAKE TYPE 3: HEAVY IMPACT (Dramatic Moment)
================================================================================

SCENARIO:
Major collision, creature roar, large explosion, significant environmental
event, or dramatic narrative moment that demands attention.

INTENSITY SPECIFICATIONS:
- Max displacement: 10-15 pixels
- Target typical: 12 pixels average
- Direction: Directional clusters with random variance

DURATION:
- Total duration: 600-800 milliseconds
- Frame count: 18-24 frames @ 30fps
- Typical: 20 frames (667ms)

FREQUENCY:
- Vibration frequency: 7 Hz
- Frame-by-frame vibration: 0.9 cycles per 667ms
- Period: 143 milliseconds per cycle
- Characteristic: Slow, heavy, powerful vibration

DISPLACEMENT PATTERN (Complex clusters):
Frames 1-3: High amplitude cluster (+12 to -12 pixels)
Frames 4-7: Medium amplitude modulation (±8 pixels)
Frames 8-12: Slow oscillation (±6 pixels)
Frames 13-18: Damping phase (±3 pixels)
Frames 19-20: Final settlement (±1 pixel)

DECAY FUNCTION:
- Decay type: Complex ease-out with multiple damping stages
- Stage 1 (0-200ms): Linear 50% reduction
- Stage 2 (200-400ms): Quadratic ease-out
- Stage 3 (400-667ms): Cubic ease-out to zero
- Formula: intensity × max(0, 1 - (frame / total_frames)^1.5)

RETURN SPEED:
- Type: Ease-in weighted motion
- Duration: 200-300 milliseconds after main shake
- Transition: Smooth deceleration over 6-9 frames
- Perception: Feels impactful and grounded

DIRECTIONAL BIAS:
- Primary bias: Direction of impact force
- Intensity: 70% toward impact direction
- Variance: 30% random orthogonal component
- Effect: Camera pushed away from impact

ROTATIONAL COMPONENT:
- Rotation type: Camera tilt
- Rotation angle: ±1-3 degrees max rotation
- Rotation sync: Follows shake intensity curve
- Duration: 80% of shake duration
- Easing: Opposite decay curve (starts zero, peaks mid-shake)

ROTATION DAMPING PROFILE:
- Peak rotation: 2 degrees at frame 10 (mid-shake)
- Frame 1: 0 degrees rotation
- Frame 10: 2.0 degrees rotation
- Frame 20: 0 degrees rotation (returned to baseline)

BLEND MODE:
- Camera offset: XY translation with rotation
- Combined effect: Significant camera displacement
- Momentum: Feels heavy and impactful

PERFORMANCE PROFILE:
- CPU cost: 0.4ms per frame (rotation calculation added)
- GPU cost: 0x (still no GPU cost)
- Memory: ~24 bytes (position + rotation)
- Frame rate impact: ~2.7% of budget

AFTERSHOCK EFFECTS:
- Primary aftershock: 0.3 seconds after main shake ends
  - Intensity: 30% of main (3.6 pixels)
  - Duration: 150ms (4-5 frames)
- Secondary aftershock: 0.7 seconds after main shake
  - Intensity: 15% of main (1.8 pixels)
  - Duration: 100ms (3 frames)
- Tertiary aftershock: 1.1 seconds after main shake
  - Intensity: 8% of main (0.96 pixels)
  - Duration: 50ms (1-2 frames)

USE CASES:

- Creature roar (presence effect)

- Dramatic impact moment

================================================================================
CAMERA DISPLACEMENT PHYSICS
================================================================================

MATHEMATICAL FORMULAS:

Base displacement calculation:
offset = intensity × sin(time × frequency × 2π) × decay_curve(time)

Where:
- intensity = max displacement in pixels
- time = elapsed time in seconds
- frequency = vibration frequency in Hz
- decay_curve = function reducing intensity over time

Decay curves by type:
- Type 1 (Linear): decay = max(0, 1 - time / duration)
- Type 2 (Ease-out): decay = max(0, 1 - (time / duration)^2)
- Type 3 (Complex): decay = max(0, 1 - (time / duration)^1.5)
- Type 4 (Extreme): decay = max(0, 1 - (time / duration)^1.2)

Random offset (per-frame):
random_offset = random(-intensity, +intensity)
actual_offset = (intensity × decay_curve) + random_offset

Rotation (for Types 3 & 4):
rotation_angle = max_rotation × sin(time × 0.3 × frequency × 2π) × decay_curve(time)

Where:
- max_rotation = maximum rotation angle in degrees
- 0.3 = rotation frequency multiplier (slower than translation)

Camera matrix transformation:
camera.position = original_position + offset_vector
camera.rotation = original_rotation + rotation_offset

PERFORMANCE & OPTIMIZATION
================================================================================

PERFORMANCE TARGETS:
- CPU cost: <1ms per frame (all shake types combined)
- GPU cost: <0.2ms per frame (color flash only)
- Memory: <100 bytes per active shake
- Frame rate: 30fps maintained without drops

OPTIMIZATION TIPS:
1. Camera shake: Pure CPU calculation (no GPU cost)
2. Color flash: Simple full-screen quad (minimal GPU cost)
3. Pooling: Pre-allocate shake event objects
4. Caching: Pre-compute sine/cosine lookup tables
5. Early exit: Disable shake calculation when decay reaches zero

MOBILE OPTIMIZATION:
- Consider reducing max intensity by 20% on mobile
- Can disable rotation component on low-end devices
- Can disable color flash on battery-saving mode
- Shake remains performant even on low-end platforms
